# 다형성
- polymorphism
- 다형성은 상속과 함께 객체지향의 중요한 특징.
- 다형성은 상속과 깊은 관계가 있다. 상속에 대한 충분한 사전 이해가 필요하다. 

__다형성은 '여러가지 형태를 가질 수 있는 능력'을 의미한다. 자바에서는 한가지 타입의 참조변수로 여러가지의 객체를 참조할 수 있도록 다형성을 프로그램적으로 구현하였다. 부모클래스 타입의 참조 변수로 자식 클래스의 객체를 참조할 수 있도록 하였다는 것이다.__

```java
tv(parent)
    ^
  extends
    |       
captionTV(chid)

tv t = new tv();
captionTV c = new captionTV();
```

지금까지는 위와 같이 객체를 생성할 때 객체 타입과 일치하는 타입의 참조변수만을 사용했다. **그러나 tv와 captionTV처럼 서로 상속관계에 있으면 부모 클래스 타입의 참조변수로 자식 클래스의 객체를 참조하도록 하는 것이 가능해진다.**
```java
    tv t = new captionTV();
```
이게 무슨 의미가 있는가? 

```java
captionTV c = new captionTV();
tv        t = new captionTV();
```

captionTV 클래스로 객체를 c,t 2개 생성했다. 참조변수 c와 t가 생성된 객체를 하나씩 참조하고 있다. 이럴 경우, **실제 객체가 captionTV타입이지만 참조변수 t로는 captionTV객체의 모든 멤버를 사용할 수는 없다.** tv타입의 참조변수로는 captionTV객체 중에서 **tv클래스의 멤버들만 사용할 수 있다.** 둘 다 같은 타입의 객체지만 참조변수의 타입에 따라 사용할 수 있는 멤버의 개수가 달라진다. 

즉 이런 코드는 불가능하다.
```java
captionTV c = new tv();
```
captionTV가 tv보다 더 큰 범위를 포함하고 있으므로 허용하지 않는다.
